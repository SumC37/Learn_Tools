# 冯诺依曼体系

- python的底层结构是C语言：用python的语法来调度封装后的C语言
- 冯诺依曼体系：
   - 硬件：提供基础指令（库和模块也是？）
   - 程序：自由组合这些指令来调度硬件实现具体功能

# 封装和接口

- 封装：手电筒的电池、灯泡等(功能的集合？)
- API(Application Programming Interface)：开关+辅助部分（接口？）
- 编程：程序按照规律调用各种API

> [!tip] 程序
> - 控制部分
>  ——API——
> - 具体功能（库和模块）

# 程序如何描述现实

* 计算机通过了解对象的属性和行为来认识对象
* API在被调用前必须转换成属性代码和行为代码

# 高效管理属性和行为

- 类是对具有相同属性和行为的类似对象的统称，它是对众多相似对象进行抽象后的结果
- python程序控制部分的五个类：
   - None
   - bool
   - int
   - float
   - string

# python

- IDE：集成开发环境，本质是在命令行中操作代码
- 浮点数：先把数变成科学计数发的表现形式，在一个标准位长上用第一位表正负，用其后一部分表指数，用最后一部分表尾数

- 函数
  - `globals()`在函数内部声明全局变量，使函数可以修改函数外部定义的变量
  - `locals()`返回一个包含当前局部变量的字典
  - `dir()`返回包含对象所有属性和方法名称的列表

- 异常处理
- 日志记录
